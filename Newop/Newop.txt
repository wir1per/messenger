Update

بذار کد را بررسی کنم تا دقیق ببینم پیش‌نمایش آخرین پیام چطور ساخته می‌شود و چرا پیام اشتباه نشان می‌دهد.
برای باگ پیام اشتباه باید ببینم پیش‌نمایش (`lastMessage`) دقیقاً موقع ارسال چطور ساخته می‌شود. بذار تابع ارسال پیام را بیرون بکشم.
بذار خود تصویر را دقیق بخوانم تا ببینم زیر اسم هر چت دقیقاً چه چیزی نوشته شده، چون باید بفهمم این ایموجی‌ها از کجا می‌آیند.
تصویر و کد را تطبیق دادم. حالا دقیق می‌دانم چه خبر است. دو موضوع جداست، بذار شفاف توضیح بدهم.

موضوع اول، پیام‌هایی که می‌گویی نفرستادی:
زیر اسم هر چت، آخرین پیامِ همان گفتگو نشان داده می‌شود، نه آخرین پیامِ تو. پس `😂😂` زیر مریم یعنی آخرین پیام آن چت این بوده (احتمالاً طرف مقابل یا دیتای تستِ قبلی فرستاده). این رفتار طبیعی است. ولی یک باگ واقعی هم آنجا هست: زیر عرفان نوشته `سیگنال`. این کلمه‌ی واقعی نیست، یک متن جایگزین (fallback) است که وقتی نوع پیام را کد نمی‌شناسد چاپ می‌شود. یعنی تابع `pushMessage` فقط متن و عکس را می‌شناسد و برای ویدیو/صدا/استیکر متن درست نمی‌سازد.

اصلاح اول؛ تابع `pushMessage` (خطوط ۲۹۵۲ تا ۲۹۵۷) را با این نسخه جایگزین کن:

```js
async function pushMessage(data) {
    const msg = { senderId: ME.uid, senderName: ME.displayName, senderIsPremium: ME.isPremium || false, createdAt: serverTimestamp(), edited: false, deleted: false, reactions: {}, readBy: [], ...data };
    await addDoc(collection(db, 'chats', activeChatId, 'messages'), msg);

    let preview;
    switch (data.type) {
        case 'image':   preview = '📷 تصویر'; break;
        case 'video':   preview = '🎬 ویدیو'; break;
        case 'voice':   preview = '🎤 پیام صوتی'; break;
        case 'sticker': preview = '🎟 استیکر'; break;
        case 'file':    preview = '📎 فایل'; break;
        case 'gif':     preview = '🖼 گیف'; break;
        default:        preview = (data.text && data.text.trim()) ? data.text : 'پیام';
    }
    if (data.isRoyalDecree) preview = '📜 سیستم: ' + preview;

    await updateDoc(doc(db, 'chats', activeChatId), { lastMessage: preview.slice(0, 45), lastTime: serverTimestamp() });
}
```

با این تغییر دیگر کلمه‌ی بی‌معنی `سیگنال` نمایش داده نمی‌شود و هر نوع پیام برچسب درست خودش را می‌گیرد. آن `😂😂` و `🎬 ویدیو` که الان روی چت‌های قدیمی هست، با همان دیتای قدیمی می‌ماند تا پیام جدیدی در آن چت رد و بدل شود؛ اگر می‌خواهی تمیز شود، آن چت‌های تستی را حذف کن.

این دو بخش را اعمال کن و آپدیت کن، بعد بهم بگو روی موبایل چطور شد. درباره‌ی «باگ‌های دیگر و چیزهای بهتر» هم بگو دقیقاً کدام بخش‌ها اذیتت می‌کنند تا یکی‌یکی جلو برویم.


بذار کد مربوط به دکمه‌ی سه‌نقطه‌ی هدر چت را پیدا کنم تا ببینم به چه پنلی وصل شده.
مشکل پیدا شد. تابع `toggleChatMenu` منو را با فرمول `left = rect.left - 180` جای می‌دهد. یعنی منو را ۱۸۰ پیکسل به سمت چپِ دکمه می‌بَرد. این فرمول برای وقتی نوشته شده که دکمه‌ی سه‌نقطه سمت راست صفحه باشد. ولی الان دکمه گوشه‌ی چپِ بالاست، پس `rect.left` عدد کوچکی است و `rect.left - 180` منفی می‌شود؛ در نتیجه منو از لبه‌ی چپِ صفحه بیرون می‌زند و همان حالتی که در عکس دور آن خط کشیدی پیش می‌آید.

راه‌حل این است که جای منو را هوشمند کنیم: اگر دکمه نزدیک لبه‌ی چپ بود، منو سمت راستِ دکمه باز شود، و در هر حال داخل صفحه بماند.

تابع `toggleChatMenu` (خطوط ۲۵۲۴ به بعد) را با این نسخه جایگزین کن:

```js
window.toggleChatMenu = function(e) {
    e.stopPropagation(); e.preventDefault();
    const m = $('chatMenu');
    const btn = e.currentTarget;
    if (m.style.display === 'block') { m.style.display = 'none'; return; }
    $('mainMenu').style.display = 'none';

    const rect = btn.getBoundingClientRect();
    m.style.display = 'block';
    m.style.position = 'fixed';

    // اول نمایش بده تا بشه عرض واقعی منو رو خوند
    const menuW = m.offsetWidth || 200;
    const menuH = m.offsetHeight || 0;
    const margin = 8;

    // افقی: تلاش کن منو رو سمت چپِ دکمه بذاری، ولی اگه از لبه بیرون زد بیارش سمت راست
    let left = rect.left - (menuW - rect.width);
    if (left < margin) left = rect.left;              // دکمه چسبیده به لبه چپه => منو رو از خودِ دکمه شروع کن
    if (left + menuW > window.innerWidth - margin)    // از لبه راست بیرون نزنه
        left = window.innerWidth - menuW - margin;

    // عمودی: زیر دکمه، ولی اگه پایین جا نبود بالای دکمه
    let top = rect.bottom + 4;
    if (top + menuH > window.innerHeight - margin)
        top = rect.top - menuH - 4;

    m.style.left = Math.max(margin, left) + 'px';
    m.style.top = Math.max(margin, top) + 'px';
};
```

این نسخه چند کار می‌کند:

- اول عرض واقعی منو را از `offsetWidth` می‌خواند، به‌جای عدد ثابت ۱۸۰.
- اگر دکمه چسبیده به لبه‌ی چپ بود (مثل الان)، منو را از خود دکمه به سمت راست باز می‌کند تا بیرون نزند.
- چک می‌کند منو نه از چپ، نه از راست و نه از پایینِ صفحه بیرون نزند و در صورت لزوم جابه‌جایش می‌کند.

یک نکته: تابع اصلی شاید چند خط دیگر هم بعد از این داشته باشد (مثلاً بستن منو با کلیک بیرون). نسخه‌ی بالا فقط بخش جای‌گذاری را امن می‌کند. اگر بعد از خط `m.style.left` در کد اصلی‌ات خطوط دیگری بود، بگو تا کاملش را ببینم و دست‌نخورده نگه دارم. اعمالش کن و روی موبایل تست کن ببین منو حالا درست زیر سه‌نقطه باز می‌شود.

بذار اول الگوی توابع موجودت رو ببینم (آپلود فایل، `switchMainTab`، نحوه‌ی کوئری Firestore و رندر) تا کد استوری دقیقاً هم‌سبک خودت باشه.
نکته‌ی مهم: جستجو نشون داد که از Firebase Storage استفاده نمی‌کنی (هیچ `uploadBytes` یا `getStorage` نبود). پس احتمالاً آواتار و رسانه‌ها رو با FileReader به base64 تبدیل می‌کنی و مستقیم تو Firestore ذخیره می‌کنی. این برای استوری مهمه چون منطق `handleStoryFile` باید با همون روش بسازم.

بذار مکانیزم واقعی آپلود رو پیدا کنم تا کد دقیق و هم‌سبک بشه.
حالا کاملاً روشنه. آپلود از طریق `uploadToImgBB(file)` انجام میشه و URL برمی‌گرده، `ME` هم آبجکت کاربره با `ME.uid`, `ME.displayName`, و آواتار از `getEffectiveAvatar()`. پس کد استوری رو دقیقاً هم‌سبک خودت می‌نویسم.

```js
// ========================= STORIES =========================
let storiesUnsub = null;
let myStories = [];          // استوری‌های خودم
let storyGroups = [];        // [{ ownerId, ownerName, ownerPhoto, items:[...] }]
let svGroupIndex = 0;        // گروه فعلی در ویوِر
let svItemIndex = 0;         // آیتم فعلی در گروه
let svTimer = null;
const STORY_DURATION = 5000; // ۵ ثانیه برای هر استوری

// --- بارگذاری استوری‌های فعال (۲۴ ساعت اخیر) ---
function loadStories() {
    if (storiesUnsub) storiesUnsub();
    const nowTs = Timestamp.now();
    const q = query(collection(db, 'stories'), where('expiresAt', '>', nowTs));
    storiesUnsub = onSnapshot(q, function(snap) {
        const groupsMap = {};
        myStories = [];
        snap.forEach(function(d) {
            const s = d.data();
            s.id = d.id;
            if (s.ownerId === ME.uid) myStories.push(s);
            if (!groupsMap[s.ownerId]) {
                groupsMap[s.ownerId] = {
                    ownerId: s.ownerId,
                    ownerName: s.ownerName || 'کاربر',
                    ownerPhoto: s.ownerPhoto || null,
                    items: []
                };
            }
            groupsMap[s.ownerId].items.push(s);
        });
        // مرتب‌سازی آیتم‌های هر گروه بر اساس زمان ساخت
        storyGroups = Object.values(groupsMap).map(function(g) {
            g.items.sort(function(a, b) {
                return (a.createdAt?.seconds || 0) - (b.createdAt?.seconds || 0);
            });
            return g;
        });
        // گروه خودم اول، بقیه بعد
        storyGroups.sort(function(a, b) {
            if (a.ownerId === ME.uid) return -1;
            if (b.ownerId === ME.uid) return 1;
            return 0;
        });
        renderStoryBar();
    }, function(err) {
        console.warn('stories load error', err);
    });
}

// --- رندر نوار استوری ---
function renderStoryBar() {

        }
    };
    bar.appendChild(addItem);

    // بقیه گروه‌ها (به جز خودم چون بالا اومد، مگر اینکه خودم استوری داشته باشم)

window.handleStoryFile = async function(event) {
    const file = event.target.files[0];
    event.target.value = '';
    if (!file) return;
    try {

            ownerName: ME.displayName || 'کاربر',
            ownerPhoto: getEffectiveAvatar() || '',
            mediaUrl: url,
            type: 'image',
            createdAt: serverTimestamp(),
            expiresAt: expires,
            viewers: []
```

```js
function escapeHtml(str) {
    if (str == null) return '';
    return String(str)
        .replace(/&/g, '&amp;').replace(/</g, '&lt;')
        .replace(/>/g, '&gt;').replace(/"/g, '&quot;')
        .replace(/'/g, '&#39;');
}
```

```js
// ============ STORIES: LOAD ============
function loadStories() {
    if (!ME) return;
    if (storiesUnsub) storiesUnsub();
    const cutoff = Timestamp.fromMillis(Date.now() - 24 * 60 * 60 * 1000);
    const q = query(
        collection(db, 'stories'),
        where('createdAt', '>', cutoff),
        orderBy('createdAt', 'desc')
    );
    storiesUnsub = onSnapshot(q, function (snap) {
        const byOwner = {};
        snap.forEach(function (d) {
            const s = d.data();
            s._id = d.id;
            if (!byOwner[s.ownerId]) {
                byOwner[s.ownerId] = {
                    ownerId: s.ownerId,
                    name: s.ownerName || 'کاربر',
                    photo: s.ownerPhoto || '',
                    items: []
                };
            }
            byOwner[s.ownerId].items.push(s);
        });
        // مرتب‌سازی آیتم‌های هر کاربر از قدیم به جدید
        Object.values(byOwner).forEach(function (g) {
            g.items.sort(function (a, b) {
                return (a.createdAt?.toMillis?.() || 0) - (b.createdAt?.toMillis?.() || 0);
            });
        });
        allStories = Object.values(byOwner);
        if (currentMainTab === 'chats') renderStoryBar();
    });
}
```

```js
// ============ STORIES: VIEWER ============
function openStoryViewer(ownerId) {
    const group = allStories.find(function (g) { return g.ownerId === ownerId; });
    if (!group || !group.items.length) return;

    storyViewerQueue = group.items.slice();
    storyIndex = 0;

    const v = $('storyViewer');
    if (!v) return;
    v.style.display = 'flex';
    document.body.style.overflow = 'hidden';

    // نام و عکس صاحب استوری در هدر ویوِر
    const head = $('storyViewerHead');
    if (head) {
        head.innerHTML =
            '<img src="' + escapeHtml(group.photo) + '">' +
            '<span>' + escapeHtml(group.name) + '</span>';
    }
    showCurrentStory();
}

function showCurrentStory() {
    if (storyTimer) { clearTimeout(storyTimer); storyTimer = null; }

    const item = storyViewerQueue[storyIndex];
    if (!item) { closeStoryViewer(); return; }

    const img = $('storyViewerImg');
    if (img) img.src = item.mediaUrl;

    // ساخت نوار پیشرفت بر اساس تعداد آیتم‌ها
    const barsWrap = $('storyBars');
    if (barsWrap) {
        let bh = '';
        for (let i = 0; i < storyViewerQueue.length; i++) {
            const cls = i < storyIndex ? 'done' : (i === storyIndex ? 'active' : '');
            bh += '<div class="story-bar-track"><div class="story-bar-fill ' + cls + '"></div></div>';
        }
        barsWrap.innerHTML = bh;
    }

    markStorySeen(item);

    // ۵ ثانیه برای هر استوری، سپس بعدی
    storyTimer = setTimeout(nextStory, 5000);
}

function nextStory() {
    storyIndex++;
    if (storyIndex >= storyViewerQueue.length) { closeStoryViewer(); return; }
    showCurrentStory();
}

function prevStory() {
    if (storyIndex <= 0) { showCurrentStory(); return; }
    storyIndex--;
    showCurrentStory();
}

function closeStoryViewer() {
    if (storyTimer) { clearTimeout(storyTimer); storyTimer = null; }
    const v = $('storyViewer');
    if (v) v.style.display = 'none';
    document.body.style.overflow = '';
    storyViewerQueue = [];
    storyIndex = 0;
    // بروزرسانی حلقه‌ها (دیده‌شده‌ها خاکستری شوند)
    if (currentMainTab === 'chats') renderStoryBar();
}

async function markStorySeen(item) {
    if (!ME || !item || !item._id) return;
    if (Array.isArray(item.viewers) && item.viewers.includes(ME.uid)) return;
    if (item.ownerId === ME.uid) return;  // استوری خودم را ثبت بازدید نکن
    try {
        await updateDoc(doc(db, 'stories', item._id), {
            viewers: arrayUnion(ME.uid)
        });
        item.viewers = item.viewers || [];
        item.viewers.push(ME.uid);
    } catch (err) {
        console.error('seen err', err);
    }
}
```

```js
let storiesUnsub = null;

function loadStories() {
    if (storiesUnsub) storiesUnsub();
    const nowTs = Timestamp.now();
    const q = query(collection(db, 'stories'), where('expiresAt', '>', nowTs));
    storiesUnsub = onSnapshot(q, function(snap) {
        const groupsMap = {};
        myStories = [];
        snap.forEach(function(d) {
            const s = d.data();
            s.id = d.id;
            if (s.ownerId === ME.uid) myStories.push(s);
            if (!groupsMap[s.ownerId]) {
                groupsMap[s.ownerId] = {
                    ownerId: s.ownerId,
                    ownerName: s.ownerName || 'کاربر',
                    ownerPhoto: s.ownerPhoto || null,
                    items: []
                };
            }
            groupsMap[s.ownerId].items.push(s);
        });
        storyGroups = Object.values(groupsMap).map(function(g) {
            g.items.sort(function(a, b) {
                return (a.createdAt?.seconds || 0) - (b.createdAt?.seconds || 0);
            });
            return g;
        });
        storyGroups.sort(function(a, b) {
            if (a.ownerId === ME.uid) return -1;
            if (b.ownerId === ME.uid) return 1;
            return 0;
        });
        renderStoryBar();
    });
```
```js
}

function buildStoryBars(count) {
    const wrap = $('storyBars');
    wrap.innerHTML = '';
    for (let i = 0; i < count; i++) {
        const seg = document.createElement('div');
        seg.className = 'sb-seg';

function showStoryItem() {
    const group = storyGroups[svGroupIndex];
    if (!group) { closeStoryViewer(); return; }
    const item = group.items[svItemIndex];
    if (!item) { nextGroup(); return; }

    buildStoryBars(group.items.length);

    // نوارهای قبلی پر، فعلی در حال پرشدن، بعدی‌ها خالی
    for (let i = 0; i < group.items.length; i++) {
        const fill = $('sbFill' + i);
        if (!fill) continue;
        if (i < svItemIndex) { fill.style.transition = 'none'; fill.style.width = '100%'; }
        else if (i > svItemIndex) { fill.style.transition = 'none'; fill.style.width = '0%'; }
    }

    $('svAvatar').src = group.ownerPhoto || '';
    $('svName').textContent = group.ownerName;
    $('svTime').textContent = storyTimeAgo(item.createdAt);
    $('storyViewerImg').src = item.mediaUrl;

    // انیمیشن نوار فعلی
    const cur = $('sbFill' + svItemIndex);
    if (cur) {
        cur.style.transition = 'none';
        cur.style.width = '0%';
        void cur.offsetWidth; // reflow
        cur.style.transition = 'width ' + (STORY_DURATION / 1000) + 's linear';
        cur.style.width = '100%';
    }

    clearTimeout(svTimer);
    svTimer = setTimeout(nextStory, STORY_DURATION);

    markStorySeen(item);
}
```

```js
async function markStorySeen(item) {
    if (item.ownerId === ME.uid) return;
    if ((item.viewers || []).includes(ME.uid)) return;
    try {
        await updateDoc(doc(db, 'stories', item.id), {
            viewers: arrayUnion(ME.uid)
        });
    } catch (e) { console.error(e); }
}

function storyTimeAgo(ts) {
    if (!ts?.seconds) return '';
    const diff = Date.now() / 1000 - ts.seconds;
    const h = Math.floor(diff / 3600);
    if (h < 1) return Math.max(1, Math.floor(diff / 60)) + ' دقیقه پیش';
    return h + ' ساعت پیش';
}
```

```js
function loadStories() {
    if (!ME) return;
    if (storiesUnsub) storiesUnsub(); // جلوگیری از اشتراک تکراری

    const cutoff = Timestamp.fromMillis(Date.now() - 24 * 60 * 60 * 1000);
    const q = query(
        collection(db, 'stories'),
        where('expiresAt', '>', Timestamp.now()),
        orderBy('expiresAt', 'asc')
    );

    storiesUnsub = onSnapshot(q, (snap) => {
        allStories = [];
        snap.forEach(d => {
            const s = d.data();
            // فیلتر ایمنی: فقط استوری‌های ساخته‌شده در ۲۴ ساعت اخیر
            if (s.createdAt && s.createdAt.toMillis() >= cutoff.toMillis()) {
                allStories.push({ id: d.id, ...s });
            }
        });
        groupStories();
        renderStoryBar();
    }, (err) => {
        console.error('loadStories error:', err);
    });
```

تابع گروه‌بندی بر اساس صاحب استوری (خودِ کاربر همیشه اول):

```js
function groupStories() {
    const map = new Map();
    allStories.forEach(s => {
        if (!map.has(s.ownerId)) {
            map.set(s.ownerId, {
                ownerId: s.ownerId,
                ownerName: s.ownerName,
                ownerPhoto: s.ownerPhoto,
                items: []
            });
        }
        map.get(s.ownerId).items.push(s);
    });

    // مرتب‌سازی استوری‌های هر کاربر بر اساس زمان ساخت
    map.forEach(g => {
        g.items.sort((a, b) => a.createdAt.toMillis() - b.createdAt.toMillis());
        // آیا همه استوری‌های این کاربر دیده شده‌اند؟
        g.allSeen = g.items.every(it =>
            Array.isArray(it.viewers) && it.viewers.includes(ME.uid)
        );
    });

    groupedStories = Array.from(map.values());

    // خودِ کاربر را به ابتدای لیست منتقل کن
    groupedStories.sort((a, b) => {
        if (a.ownerId === ME.uid) return -1;
        if (b.ownerId === ME.uid) return 1;
        if (a.allSeen !== b.allSeen) return a.allSeen ? 1 : -1; // دیده‌نشده‌ها جلوتر
        return 0;
    });
}
```

```js
function escapeHtml(str) {

    }[c]));
```

```js
    clearTimeout(storyTimer);

    // عکس
    const img = $('storyViewerImg');
    if (img) img.src = item.mediaUrl;

function buildStoryBars() {
    const wrap = $('storyBars');
    if (!wrap) return;
    let html = '';
    for (let i = 0; i < storyViewerQueue.length; i++) {
        let fillClass = '';

function storyTimeAgo(ts) {
    if (!ts) return '';

    const min = Math.floor(ms / 60000);
    if (min < 60) return `${min} دقیقه پیش`;
    const hr = Math.floor(min / 60);
```

```js
    if (!item || !ME) return;

    if (Array.isArray(item.viewers) && item.viewers.includes(ME.uid)) return; // قبلاً دیده

}
```

### `loadStories()` — دریافت زنده + گروه‌بندی
```js
function loadStories() {
    if (!ME) return;
    if (storiesUnsub) storiesUnsub(); // listener قبلی رو ببند

    // فقط استوری‌هایی که هنوز منقضی نشدن
    const q = query(
        collection(db, 'stories'),
        where('expiresAt', '>', Timestamp.now()),
        orderBy('expiresAt', 'asc')
    );

    storiesUnsub = onSnapshot(q, (snap) => {
        allStories = [];
        snap.forEach(d => {
            allStories.push({ id: d.id, ...d.data() });
        });

        // گروه‌بندی بر اساس صاحب استوری
        groupedStories = {};
        allStories.forEach(s => {
            if (!groupedStories[s.ownerId]) {
                groupedStories[s.ownerId] = {
                    name: s.ownerName || 'کاربر',
                    photoURL: s.ownerPhoto || '',
                    items: []
                };
            }
            groupedStories[s.ownerId].items.push(s);
        });

        // مرتب‌سازی استوری‌های هر گروه بر اساس زمان ساخت (قدیمی → جدید)
        Object.values(groupedStories).forEach(g => {
            g.items.sort((a, b) => {
                const ta = a.createdAt?.toMillis?.() || 0;
                const tb = b.createdAt?.toMillis?.() || 0;
                return ta - tb;
            });
        });

        // فقط اگه تب چت‌ها بازه نوار رو رفرش کن
        if (currentMainTab === 'chats') renderStoryBar();
    }, (err) => {
        console.error('loadStories error:', err);
    });
}
```

```js
function showStoryItem() {
    clearTimeout(storyTimer);

    const item = storyViewerItems[storyIndex];
    if (!item) { closeStoryViewer(); return; }

    // عکس
    $('storyViewerImg').src = item.mediaUrl;

}

function nextStory() {
    clearTimeout(storyTimer);
    if (storyIndex < storyViewerItems.length - 1) {
        storyIndex++;
        showStoryItem();
    } else {
        closeStoryViewer();
    }
}

function prevStory() {
    clearTimeout(storyTimer);
    if (storyIndex > 0) {
        storyIndex--;
        showStoryItem();
    } else {
        showStoryItem(); // از اول همین استوری
    }
}

    clearTimeout(storyTimer);
    const viewer = $('storyViewer');
    viewer.classList.remove('show');

    storyIndex = 0;
```

```js
function storyTimeAgo(ts) {
    if (!ts || !ts.toMillis) return 'همین حالا';
    const diff = Date.now() - ts.toMillis();
    const min = Math.floor(diff / 60000);
    if (min < 1) return 'همین حالا';
    if (min < 60) return `${min} دقیقه پیش`;
    const hr = Math.floor(min / 60);
    return `${hr} ساعت پیش`;
}
```

اگه `escapeHtml` توی فایل نداری، این مینیمالش هست:

```js
function escapeHtml(str) {
    return String(str || '')
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;')
        .replace(/"/g, '&quot;');
}
```

حالا کل چرخه کامله: آپلود ← لیست زنده ← نوار ← ویوِر با نوار پیشرفت ← ثبت بازدید ← خاموش‌شدن حلقه. می‌خوای تست سریع منطق دیده‌شدن (seen) رو با هم چک کنیم یا قانون امنیتی Firestore برای کالکشن `stories` رو هم برات بنویسم؟


بذار اول ساختار واقعی کالکشن `stories` و فیلدهاش رو از فایل پروژه ببینم تا قانون امنیتی دقیقاً با همون شکل داده‌ها بخونه، نه حدسی.
فایل نشون داد که کد Stories هنوز به پروژه اضافه نشده. پس باهوشانه‌ترین کار اینه که قانون Firestore رو **بر اساس همون ساختار داده‌ای که با هم طراحی کردیم** بنویسم.

---

### `firestore.rules` — قانون کالکشن Stories

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // ── کاربران ──────────────────────────────────────
    match /users/{uid} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == uid;
    }

    // ── چت‌ها ─────────────────────────────────────────
    match /chats/{chatId} {
      allow read, write: if request.auth != null
        && request.auth.uid in resource.data.members;

      match /messages/{msgId} {
        allow read, write: if request.auth != null
          && request.auth.uid in get(/databases/$(database)/documents/chats/$(chatId)).data.members;
      }
    }

    // ── استوری‌ها ─────────────────────────────────────
    match /stories/{storyId} {

      // هر لاگین‌کرده‌ای می‌تونه استوری‌های زنده رو بخونه
      allow read: if request.auth != null;

      // فقط صاحب استوری می‌تونه ایجاد کنه
      allow create: if request.auth != null
        && request.auth.uid == request.resource.data.ownerId
        && request.resource.data.keys().hasAll(['ownerId','imageUrl','createdAt','expiresAt'])
        && request.resource.data.viewers is list
        && request.resource.data.expiresAt is timestamp;

      // فقط دو عملیات مجاز روی استوری موجود:
      // ۱) صاحب استوری حذف کنه
      // ۲) هر کاربری uid خودش رو به viewers اضافه کنه (markStorySeen)
      allow update: if request.auth != null
        && (
          request.auth.uid == resource.data.ownerId
          ||
          (
            request.resource.data.diff(resource.data).affectedKeys().hasOnly(['viewers'])
            && request.resource.data.viewers.size() == resource.data.viewers.size() + 1
            && request.auth.uid in request.resource.data.viewers
          )
        );

      allow delete: if request.auth != null
        && request.auth.uid == resource.data.ownerId;
    }
  }
}
```

---

### چرا این قانون‌ها درسته؟

| عملیات | مجاز برای |
|---|---|
| `read` | هر کاربر لاگین‌کرده |
| `create` | فقط اگه `ownerId == uid` خودته + فیلدهای اجباری موجود باشن |
| `update` | صاحب استوری (هر چیزی) **یا** هر کس فقط `viewers` رو +۱ کنه |
| `delete` | فقط صاحب استوری |

نکته مهم: قانون `update` برای `viewers` دقیقاً با `arrayUnion` در `markStorySeen` هماهنگه — نمی‌ذاره کسی `viewers` رو کم کنه یا چیز دیگه‌ای تغییر بده.

---

این فایل رو در ریشه پروژه کنار `firebase.json` بذار و با `firebase deploy --only firestore:rules` دیپلوی کن.

باشه داداش، بذار اول دقیق ببینم ساختار فعلی فایل چطوریه تا کد رو طوری بنویسم که دقیق بهش بچسبه (مخصوصاً `onAuthStateChanged`، `switchMainTab` و جایی که `mainList` هست).
خب داداش، حالا دقیق می‌دونم باید کجا وصلش کنم. بریم سراغ کد کامل. سه بخش می‌فرستم: **JS**، **HTML**، **CSS**. بعدش هم بهت می‌گم دقیقاً کجا بذاریشون.

## ۱) کد JavaScript

این رو یه جای مناسب توی بلوک `<script type="module">` (مثلاً قبل از `onAuthStateChanged`) اضافه کن:

```js
/* ===================== STORIES ===================== */
let storiesUnsub = null;
let storyGroups = [];      // [{ ownerId, ownerName, ownerPhoto, items:[], allSeen }]
let svGroupIndex = 0;
let svItemIndex = 0;
let svTimer = null;
const STORY_DURATION = 5000;

// لود استوری‌ها به صورت زنده
function loadStories() {
    if (!ME) return;
    if (storiesUnsub) storiesUnsub();

    const q = query(
        collection(db, 'stories'),
        where('expiresAt', '>', Timestamp.now()),
        orderBy('expiresAt', 'asc')
    );

    storiesUnsub = onSnapshot(q, (snap) => {
        const all = [];
        snap.forEach(d => all.push({ id: d.id, ...d.data() }));

        // گروه‌بندی بر اساس صاحب استوری
        const map = {};
        all.forEach(s => {
            if (!map[s.ownerId]) {
                map[s.ownerId] = {
                    ownerId: s.ownerId,
                    ownerName: s.ownerName || 'کاربر',
                    ownerPhoto: s.ownerPhoto || '',
                    items: []
                };
            }
            map[s.ownerId].items.push(s);
        });

        storyGroups = Object.values(map).map(g => {
            g.items.sort((a, b) => {
                const ta = a.createdAt?.seconds || 0;
                const tb = b.createdAt?.seconds || 0;
                return ta - tb;
            });
            g.allSeen = g.items.every(it => (it.viewers || []).includes(ME.uid));
            return g;
        });

        // استوری خودم همیشه اول
        storyGroups.sort((a, b) => {
            if (a.ownerId === ME.uid) return -1;
            if (b.ownerId === ME.uid) return 1;
            if (a.allSeen !== b.allSeen) return a.allSeen ? 1 : -1;
            return 0;
        });

        renderStoryBar();
    }, (err) => {
        console.error('loadStories error:', err);
    });
}

// رندر نوار استوری
function renderStoryBar() {
    const bar = $('storyBar');
    if (!bar || !ME) return;

    let html = '';

    // آیتم «استوری من» با دکمه +

    const myPhoto = ME.photoURL || '';
    html += `
        <div class="story-item" onclick="${myGroup ? `openStoryViewer('${ME.uid}')` : 'pickStoryFile()'}">
            <div class="story-ring add ${myGroup && !myGroup.allSeen ? '' : 'seen'}">
                <img class="story-avatar" src="${escapeHtml(myPhoto)}" alt="">
                <div class="story-add-badge" onclick="event.stopPropagation(); pickStoryFile()">+</div>
            </div>
            <div class="story-name">استوری من</div>
        </div>
    `;

    // بقیه کاربرها
    storyGroups.forEach(g => {
        if (g.ownerId === ME.uid) return;
        html += `
            <div class="story-item" onclick="openStoryViewer('${g.ownerId}')">
                <div class="story-ring ${g.allSeen ? 'seen' : ''}">
                    <img class="story-avatar" src="${escapeHtml(g.ownerPhoto)}" alt="">
                </div>
                <div class="story-name">${escapeHtml(g.ownerName)}</div>
            </div>
        `;
    });

    bar.innerHTML = html;
}

// باز کردن انتخاب فایل

// آپلود استوری
window.handleStoryFile = async function(event) {
    const file = event.target.files && event.target.files[0];
    event.target.value = '';
    if (!file) return;

    if (!file.type.startsWith('image/')) {
        showMockModal('فقط عکس می‌تونی استوری بذاری.');
        return;
    }

    try {
        showMockModal('در حال آپلود استوری...');
        const url = await uploadToImgBB(file);
        if (!url) { showMockModal('آپلود ناموفق بود. دوباره تلاش کن.'); return; }

        const expires = Timestamp.fromMillis(Date.now() + 24 * 60 * 60 * 1000);
        await addDoc(collection(db, 'stories'), {
            ownerId: ME.uid,
            ownerName: ME.displayName || 'کاربر',
            ownerPhoto: ME.photoURL || '',
            mediaUrl: url,
            type: 'image',
            createdAt: serverTimestamp(),
            expiresAt: expires,
            viewers: []
        });

        showMockModal('استوری شما با موفقیت ثبت شد.');
    } catch (e) {

        showMockModal('خطا در ثبت استوری.');
    }
};

// باز کردن viewer
window.openStoryViewer = function(ownerId) {
    const idx = storyGroups.findIndex(g => g.ownerId === ownerId);
    if (idx === -1) return;
    svGroupIndex = idx;

    // اولین آیتم دیده‌نشده، وگرنه اولی
    const items = storyGroups[idx].items;
    const firstUnseen = items.findIndex(it => !(it.viewers || []).includes(ME.uid));
    svItemIndex = firstUnseen === -1 ? 0 : firstUnseen;

    $('storyViewer').classList.add('open');
    showStoryItem();
};

// نمایش آیتم فعلی
function showStoryItem() {
    const group = storyGroups[svGroupIndex];
    if (!group) { closeStoryViewer(); return; }
    const item = group.items[svItemIndex];
    if (!item) { closeStoryViewer(); return; }

    $('svAvatar').src = group.ownerPhoto || '';
    $('svName').textContent = group.ownerName || 'کاربر';
    $('svTime').textContent = storyTimeAgo(item.createdAt);
    $('storyViewerImg').src = item.mediaUrl || '';

    buildStoryBars();
    markStorySeen(item);

    if (svTimer) clearTimeout(svTimer);
    svTimer = setTimeout(nextStory, STORY_DURATION);
}

// ساخت نوارهای پیشرفت
function buildStoryBars() {
    const group = storyGroups[svGroupIndex];
    const bars = $('storyBars');
    if (!bars || !group) return;

    let html = '';
    group.items.forEach((_, i) => {
        let fill = '0%';
        if (i < svItemIndex) fill = '100%';
        html += `
            <div class="sb-seg">
                <div class="sb-fill" style="width:${fill};${i === svItemIndex ? `animation: sbGrow ${STORY_DURATION}ms linear forwards;` : ''}"></div>
            </div>
        `;
    });
    bars.innerHTML = html;
}

// استوری بعدی
window.nextStory = function() {
    const group = storyGroups[svGroupIndex];
    if (!group) { closeStoryViewer(); return; }

    if (svItemIndex < group.items.length - 1) {
        svItemIndex++;
        showStoryItem();
    } else if (svGroupIndex < storyGroups.length - 1) {
        svGroupIndex++;
        svItemIndex = 0;
        showStoryItem();
    } else {
        closeStoryViewer();
    }
};

// استوری قبلی
window.prevStory = function() {
    if (svItemIndex > 0) {
        svItemIndex--;
        showStoryItem();
    } else if (svGroupIndex > 0) {
        svGroupIndex--;
        svItemIndex = storyGroups[svGroupIndex].items.length - 1;
        showStoryItem();
    }
};

// بستن viewer
window.closeStoryViewer = function() {
    if (svTimer) clearTimeout(svTimer);
    svTimer = null;
    $('storyViewer').classList.remove('open');
};

// ثبت بازدید
async function markStorySeen(item) {
    if (!item || !ME) return;
    if ((item.viewers || []).includes(ME.uid)) return;
    try {
        await updateDoc(doc(db, 'stories', item.id), {
            viewers: arrayUnion(ME.uid)
        });
        item.viewers = [...(item.viewers || []), ME.uid];
    } catch (e) {
        console.error('markStorySeen error:', e);
    }
}

// زمان نسبی
function storyTimeAgo(ts) {
    if (!ts || !ts.seconds) return '';
    const diff = Date.now() - ts.seconds * 1000;
    const min = Math.floor(diff / 60000);
    if (min < 1) return 'همین الان';
    if (min < 60) return `${min} دقیقه پیش`;
    const hr = Math.floor(min / 60);
    return `${hr} ساعت پیش`;
}
```

## جمع‌بندی جای‌گذاری

- JS: داخل `<script type="module">` قبل از `onAuthStateChanged`
- خط `loadStories()` بعد از `ME = snap.data()`
- `renderStoryBar()` توی شاخه `chats` تابع `switchMainTab`
- `#storyBar` بین `#listTitle` و `#mainList`
- بلوک `input` و `#storyViewer` قبل از `</body>`
- CSS ته `<style>`

یه نکته: کد از توابعی مثل `query`, `where`, `orderBy`, `addDoc`, `updateDoc`, `arrayUnion`, `Timestamp`, `serverTimestamp` و هلپر `escapeHtml` استفاده می‌کنه. اگه هرکدوم توی importهای Firebase پروژه‌ت نبود بهم بگو تا چکش کنیم. می‌خوای importها رو برات بررسی کنم؟

خبر خوب، تقریباً همه importهای مورد نیازِ Firestore توی پروژه موجوده. بذار `escapeHtml` رو هم چک کنم چون توی رندر استوری‌بار ازش استفاده می‌کنیم.
نتیجه بررسی:

importهای Firestore — کاملاً اوکیه. این خط همه‌چیزی که لازم داریم رو داره:

```js
import { getFirestore, doc, getDoc, setDoc, updateDoc, collection, addDoc, query, where, orderBy, onSnapshot, serverTimestamp, getDocs, limit, increment, arrayUnion, arrayRemove, deleteDoc, Timestamp } from "...firebase-firestore.js";
```

پس `query`, `where`, `orderBy`, `addDoc`, `updateDoc`, `arrayUnion`, `Timestamp`, `serverTimestamp`, `onSnapshot` همه موجودن. مشکلی نداریم.

هلپرها:
- `$(id)` موجوده (خط ۲۲۴۹) ✅
- `showMockModal` موجوده (خط ۲۳۷۳) ✅
- `uploadToImgBB` موجوده (خط ۲۳۵۸) ✅
- `escapeHtml` موجود نیست ❌

پس فقط یک کار مونده: چون `escapeHtml` توی پروژه نیست، باید خودمون اضافه‌اش کنیم وگرنه `renderStoryBar()` ارور می‌ده.

این تابع کوچیک رو یه جای امن (مثلاً درست بعد از تعریف `$` در خط ۲۲۴۹) اضافه کن:

```js
function escapeHtml(str) {
    if (str == null) return '';
    return String(str)
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;')
        .replace(/"/g, '&quot;')
        .replace(/'/g, '&#39;');
}
```

با این اضافه‌شدن، کل کد استوری بدون هیچ وابستگیِ گمشده‌ای کار می‌کنه.

می‌خوای کل کد JavaScript استوری (همون `loadStories`, `renderStoryBar`, `handleStoryFile`, `openStoryViewer` و بقیه) رو یکجا و تمیز برات بفرستم که فقط کپی‌پیست کنی؟

میرم ساختار فعلی استوری ویوِر، چت و حضور آنلاین رو از فایل دربیارم تا کدها رو دقیق وصل کنم.
خوبه، حالا وضعیت روشنه. کد استوری توی فایل نیست پس از صفر می‌سازیمش، ولی چت و presence پایه دارن.

برای اینکه ویس‌مسیج و وضعیت آنلاین رو دقیق به همون ساختار فعلی وصل کنم (نه اینکه چیز موازی بسازم)، باید بدنه کامل `sendMessage`، HTML نوار ورودی چت، و کد فعلی presence/typing رو ببینم. بذار اینا رو دربیارم:
بذار دقیقاً همون رنج‌هایی که نیازه رو دربیارم تا کد رو روی ساختار واقعی بنویسم.
عالی، حالا ساختار واقعی رو دارم. شروع می‌کنم از **ویس‌مسیج** چون کاملاً خودبسنده‌ست و همه قلاب‌هاش (`#inputBar`، `pushMessage`، `renderMessage`) رو الان دقیق می‌شناسم.

نکته مهم قبل از کد: ویس باید جایی ذخیره بشه. توی کدت تصویر چطوری آپلود میشه؟ چون باید ببینم Firebase Storage داری یا تصاویر رو base64 توی Firestore می‌ریزی. این تعیین می‌کنه ویس رو هم همونجوری ذخیره کنم که هماهنگ باشه.

فعلاً کد ویس‌مسیج رو طوری می‌نویسم که با **Firebase Storage** کار کنه (روش استاندارد و سبک برای صدا). اگه Storage نداری بگو تا نسخه base64 بدم.

## ۳. JavaScript — منطق ضبط، آپلود و پخش

این بلوک رو نزدیک بقیه توابع چت اضافه کن:

```js
// ===== Voice Messages =====
let mediaRecorder = null, voiceChunks = [], voiceStartTime = 0, voiceTimerInt = null, voiceCancelled = false;

window.startVoiceRecord = async function(e){
    if (e) e.preventDefault();
    if (!activeChatId) return;
    voiceCancelled = false;
    try {
        const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
        mediaRecorder = new MediaRecorder(stream);
        voiceChunks = [];
        mediaRecorder.ondataavailable = ev => { if (ev.data.size > 0) voiceChunks.push(ev.data); };
        mediaRecorder.onstop = handleVoiceStop;
        mediaRecorder.start();
        voiceStartTime = Date.now();
        $('voiceRecordBar').classList.add('show');
        voiceTimerInt = setInterval(updateVoiceTimer, 200);
    } catch (err) {
        toast('دسترسی به میکروفون رد شد', '⚠️');
    }
};

function updateVoiceTimer(){
    const s = Math.floor((Date.now() - voiceStartTime) / 1000);
    $('voiceTimer').textContent = Math.floor(s/60) + ':' + String(s%60).padStart(2,'0');
}

window.stopVoiceRecord = function(e){
    if (e) e.preventDefault();
    if (!mediaRecorder || mediaRecorder.state === 'inactive') return;
    clearInterval(voiceTimerInt);
    $('voiceRecordBar').classList.remove('show');
    mediaRecorder.stop();
    mediaRecorder.stream.getTracks().forEach(t => t.stop());
};

window.cancelVoiceRecord = function(e){
    if (e) e.preventDefault();
    voiceCancelled = true;
    if (mediaRecorder && mediaRecorder.state !== 'inactive') {
        clearInterval(voiceTimerInt);
        $('voiceRecordBar').classList.remove('show');
        mediaRecorder.stop();
        mediaRecorder.stream.getTracks().forEach(t => t.stop());
    }
};

async function handleVoiceStop(){
    const duration = Math.round((Date.now() - voiceStartTime) / 1000);
    if (voiceCancelled || duration < 1 || voiceChunks.length === 0) return;
    const blob = new Blob(voiceChunks, { type: 'audio/webm' });

    // آپلود به Firebase Storage
    const path = `voices/${activeChatId}/${ME.uid}_${Date.now()}.webm`;
    const sRef = storageRef(storage, path);
    try {
        toast('در حال ارسال ویس...', '🎤');
        await uploadBytes(sRef, blob);
        const url = await getDownloadURL(sRef);
        await pushMessage({ type: 'voice', voiceUrl: url, voiceDuration: duration });
    } catch (err) {
        toast('خطا در ارسال ویس', '⚠️');
    }
}

window.playVoice = function(btn, url){
    let audio = btn._audio;
    if (!audio) {
        audio = new Audio(url);
        btn._audio = audio;
        const prog = btn.parentElement.querySelector('.vm-prog');
        const timeEl = btn.parentElement.querySelector('.vm-time');
        audio.ontimeupdate = () => {
            if (audio.duration) prog.style.width = (audio.currentTime/audio.duration*100) + '%';
            const s = Math.floor(audio.currentTime);
            timeEl.textContent = Math.floor(s/60)+':'+String(s%60).padStart(2,'0');
        };
        audio.onended = () => { btn.textContent = '▶'; prog.style.width = '0%'; };
    }
    if (audio.paused) { audio.play(); btn.textContent = '⏸'; }
    else { audio.pause(); btn.textContent = '▶'; }
};
```

دو تا چیز رو تأیید کن تا بقیه دقیق بشه:

1. **Firebase Storage داری؟** اگه نه، بگو تا نسخه base64 ویس بدم (محدودیت حجم داره ولی بدون Storage کار می‌کنه).

بعدش میریم سراغ **وضعیت آنلاین زنده** و بعد **استوری‌ها (حذف/بازدیدکننده/ری‌اکشن)**.

می‌رم سراغ **وضعیت آنلاین زنده**. اول باید ببینم هدر چت کجا و چطور رندر می‌شه و `lastOnline` کجا نمایش داده می‌شه تا کد دقیقاً سرجاش بشینه.
ساختار رو دارم. مشکل فعلی واضحه: `renderChatHeader()` فقط **یک‌بار** موقع `openChat` اجرا می‌شه، پس وضعیت آنلاین طرف مقابل **زنده نیست** — تا چت رو نبندی و باز نکنی، آپدیت نمی‌شه. همینطور خود تو فقط هر ۶۰ ثانیه `lastOnline` رو می‌نویسی.

راه‌حل: یک `onSnapshot` روی داکیومنت کاربر طرف مقابل می‌ذارم تا هر تغییر `lastOnline` بلافاصله توی `#hSub` و نقطه آنلاین منعکس بشه.

## ۱. JS — اشتراک زنده وضعیت طرف مقابل

این بلوک رو کنار بقیه توابع چت اضافه کن:

```js
// ===== Live presence of chat partner =====
let partnerStatusUnsub = null;
let partnerTickInt = null;

function subscribePartnerStatus() {
    // پاک‌سازی اشتراک قبلی
    if (partnerStatusUnsub) { partnerStatusUnsub(); partnerStatusUnsub = null; }
    if (partnerTickInt) { clearInterval(partnerTickInt); partnerTickInt = null; }

    if (!activeChat || activeChat.type !== 'private') return;
    const other = activeChat.members.find(m => m !== ME.uid);
    if (!other) return;

    partnerStatusUnsub = onSnapshot(doc(db, 'users', other), snap => {
        if (!snap.exists()) return;
        applyPartnerStatus(snap.data().lastOnline);
    });

    // رفرش متن «X دقیقه پیش» هر ۳۰ ثانیه بدون خواندن مجدد از سرور
    partnerTickInt = setInterval(() => {
        if (partnerLastOnline) applyPartnerStatus(partnerLastOnline);
    }, 30000);
}

let partnerLastOnline = null;
function applyPartnerStatus(lastOnline) {
    partnerLastOnline = lastOnline;
    const txt = getLastSeenText(lastOnline);
    $('hSub').textContent = txt;
    $('hSub').style.color = txt === 'آنلاین' ? 'var(--green)' : 'var(--gold)';
}
```

## ۲. وصل کردن به `openChat`

داخل `openChat` (خط ۲۷۲۷)، بعد از `subscribeTyping(chatId);` این رو اضافه کن:

```js
subscribePartnerStatus();
```

و وقتی چت بسته می‌شه یا چت دیگه‌ای باز می‌شه، اشتراک قبلی باید پاک بشه. چون `subscribePartnerStatus()` خودش اول `partnerStatusUnsub()` رو صدا می‌زنه، با هر بار `openChat` تمیز می‌شه. ولی برای بستن کامل چت، اگه تابع بستن داری (مثل `closeChat`)، این رو هم توش بذار:

```js
if (partnerStatusUnsub) { partnerStatusUnsub(); partnerStatusUnsub = null; }
if (partnerTickInt) { clearInterval(partnerTickInt); partnerTickInt = null; }
```

## ۳. آنلاین‌تر کردن خودت (مهم)

الان `updatePresence` هر ۶۰ ثانیه اجرا می‌شه، یعنی طرف مقابل ممکنه تا یک دقیقه بعد از خروجت همچنان «آنلاین» ببینتت. دو بهبود:

**الف) رفرش سریع‌تر** — توی `enterApp` بازه رو از `60000` به `30000` کم کن:

```js
presenceInterval = setInterval(updatePresence, 30000);
```

**ب) آپدیت روی فعالیت و خروج** — این‌ها رو یک‌بار توی `enterApp` اضافه کن:

```js
// آپدیت presence هنگام برگشتن به تب
document.addEventListener('visibilitychange', () => {
    if (!document.hidden) updatePresence();
});
```

`getLastSeenText` چون آستانه‌ی «آنلاین» داره (احتمالاً زیر ۱-۲ دقیقه)، با رفرش ۳۰ ثانیه‌ای طرف مقابل، انتقال از آنلاین به آفلاین نرم و دقیق می‌شه.

---

یک نکته که باید چک کنم تا کامل بشه: آستانه‌ی «آنلاین» توی `getLastSeenText` چنده؟ یعنی چند ثانیه/دقیقه `lastOnline` رو آنلاین حساب می‌کنه؟ این تعیین می‌کنه بازه رفرش ۳۰ ثانیه کافیه یا نه. می‌خوای بدنه‌ی `getLastSeenText` رو دربیارم تا آستانه رو ببینم و هماهنگش کنم؟

بعدش می‌ریم سراغ **استوری‌ها (حذف دستی، لیست بازدیدکننده، ری‌اکشن/ریپلای)**.


بریم سراغ بعدی. طبق روند، نوبت قابلیت‌های پیشرفته‌ی استوری بود: حذف دستی، لیست بازدیدکننده‌ها، و ری‌اکشن/ریپلای روی استوری. اول بذار وضعیت دقیق فعلی فایل رو از نظر استوری و viewerش چک کنم تا روی همون ساختار سوار کنم.
تأیید شد: کد استوری با این نام‌ها توی فایل نیست، پس استوری از صفر ساخته می‌شه (همون طراحی که آماده کردیم). و `getLastSeenText` هست ولی آستانه‌ی دقیقش هنوز معلوم نیست. بذار همون چند خط رو دربیارم تا آستانه‌ی «آنلاین» رو ببینم — هم برای هماهنگی بازه‌ی ۳۰ ثانیه، هم چون استوری به همین منطق ربطی نداره ولی presence مهمه.
آستانه‌ی «آنلاین» مشخص شد: **زیر ۱۲۰۰۰۰ms یعنی ۲ دقیقه**. پس تنظیم presence ما درسته — با رفرش ۳۰ ثانیه‌ای، طرف مقابل حداکثر تا ۲ دقیقه بعد از خروج آفلاین می‌شه و انتقالش نرم و دقیقه. هیچ تغییری توی `getLastSeenText` لازم نیست.

حالا بریم سراغ قابلیت‌های پیشرفته‌ی استوری. این‌ها روی همون ساختار استوری که از صفر طراحی کردیم سوار می‌شن (`stories` collection با فیلدهای `viewers`, `ownerId`, `mediaUrl` و ...).

## ۳. ری‌اکشن و ریپلای روی استوری

استوری دیگران یک ورودی ته صفحه داره: تایپ جواب + دکمه‌های ری‌اکشن سریع. جواب به‌صورت پیام خصوصی به صاحب استوری می‌ره.

```js
// ری‌اکشن سریع روی استوری → به‌عنوان پیام به صاحب استوری
async function reactToStory(emoji) {
    const item = svItems[svItemIndex];
    if (!item || item.ownerId === ME.uid) return;
    await sendStoryReply(emoji, item);
    // فیدبک کوچک
    floatEmoji(emoji);
}

async function sendStoryReply(text, item) {
    item = item || svItems[svItemIndex];
    if (!item || !text.trim() || item.ownerId === ME.uid) return;

    // پیدا/ساخت چت خصوصی با صاحب استوری
    const chatId = await getOrCreatePrivateChat(item.ownerId);

    await pushMessage(chatId, {
        text: text.trim(),
        storyReplyTo: {
            storyId: item.id,
            mediaUrl: item.mediaUrl,
            ownerName: item.ownerName
        }
    });

    $('svReplyInput').value = '';
    showMockModal('جواب فرستاده شد');
}
```

تابع کمکی برای انیمیشن ری‌اکشن:

```js
function floatEmoji(emoji) {
    const el = document.createElement('div');
    el.className = 'story-float-emoji';
    el.textContent = emoji;
    $('storyViewer').appendChild(el);
    setTimeout(() => el.remove(), 1200);
}
```

---

دو تا نکته‌ی وابستگی که باید چک کنم تا این بخش کامل و بدون باگ بشه:

1. **`getOrCreatePrivateChat`** — توی فایل هست یا با اسم دیگه‌ای پیاده شده؟ چون `sendStoryReply` بهش وابسته‌ست.
2. **امضای دقیق `pushMessage`** — آیا `pushMessage(chatId, obj)` می‌گیره یا فقط `pushMessage(obj)` و خودش `activeChatId` رو می‌خونه؟ این روی نحوه‌ی صدا زدنش اثر داره.

می‌خوای این دو تا رو از فایل دربیارم تا امضاها رو دقیق هماهنگ کنم؟ بعدش CSS مربوط به دکمه حذف، شیت بازدیدکننده‌ها و ری‌اکشن استوری رو هم می‌دم تا کل بخش استوری بسته بشه.

می‌بررسم امضای دقیق توابع رو تا کد بدون باگ بسته بشه.
امضاها رو دقیق درآوردم. دو تا نکته مهم که کد رو شکل می‌ده:

- `pushMessage(data)` فقط یک آرگومان می‌گیره و خودش `activeChatId` رو می‌خونه. پس برای جواب استوری نمی‌تونم چت مقصد رو پاس بدم؛ باید مستقیم با `addDoc` بفرستم تا چتِ بازِ فعلی به‌هم نخوره.
- `getOrCreatePrivateChat` و `DEFAULT_AVATAR` و `floatEmoji`/`pauseStory` توی فایل **نیستن** — همه رو خودم می‌سازم.

این هم بسته‌ی نهایی استوری. سه بخش: JS، HTML، CSS.

## ۱) JavaScript — این‌ها رو کنار بقیه‌ی توابع استوری بذار

```js
    '<svg xmlns="http://www.w3.org/2000/svg" width="80" height="80"><rect width="80" height="80" rx="40" fill="#2a2a3a"/><text x="50%" y="55%" font-size="34" text-anchor="middle" fill="#888">👤</text></svg>'

// مکث/ادامه‌ی استوری (برای وقتی شیت بازدید یا ورودی جواب باز می‌شه)
function pauseStory() {
    clearTimeout(svTimer);
    const fill = document.querySelector('#storyBars .sb-fill.active');
    if (fill) fill.style.animationPlayState = 'paused';
}
function resumeStory() {
    const fill = document.querySelector('#storyBars .sb-fill.active');
    if (fill) {
        fill.style.animationPlayState = 'running';
        // زمان باقی‌مانده رو از روی عرض فعلی نوار حساب می‌کنیم
        const w = fill.getBoundingClientRect().width;
        const full = fill.parentElement.getBoundingClientRect().width || 1;
        const remain = Math.max(0, STORY_DURATION * (1 - w / full));
        clearTimeout(svTimer);
        svTimer = setTimeout(nextStory, remain);
    } else {
        clearTimeout(svTimer);
        svTimer = setTimeout(nextStory, STORY_DURATION);
    }
}

// انیمیشن شناور ری‌اکشن

// ---------- جواب و ری‌اکشن استوری ----------
// چت خصوصی با یک کاربر رو پیدا یا می‌سازه و idش رو برمی‌گردونه
async function getOrCreatePrivateChat(otherUid) {
    // جست‌وجو بین چت‌هایی که من عضوشونم
    const q = query(collection(db, 'chats'), where('members', 'array-contains', ME.uid));
    const snap = await getDocs(q);
    let found = null;
    snap.forEach(d => {
        const c = d.data();
        if (c.type === 'private' && Array.isArray(c.members)
            && c.members.length === 2 && c.members.includes(otherUid)) {
            found = d.id;
        }
    });
    if (found) return found;
    // ساخت چت جدید
    const ref = await addDoc(collection(db, 'chats'), {
        type: 'private',
        members: [ME.uid, otherUid],
        photoURL: '',
        lastMessage: '',
        lastTime: serverTimestamp(),
        background: ''
    });
    return ref.id;
}

// فرستادن پیام مستقیم به یک چت بدون باز کردنش (تا استوری بسته نشه)
async function sendDirectMessage(chatId, data) {
    const msg = {
        senderId: ME.uid,
        senderName: ME.displayName,
        senderIsPremium: ME.isPremium || false,
        createdAt: serverTimestamp(),
        edited: false, deleted: false,
        reactions: {}, readBy: [],
        ...data
    };
    await addDoc(collection(db, 'chats', chatId, 'messages'), msg);
    const preview = data.text || 'پاسخ استوری';
    await updateDoc(doc(db, 'chats', chatId), {
        lastMessage: preview.slice(0, 45),
        lastTime: serverTimestamp()
    });
}

async function sendStoryReply(text) {
    const item = svItems[svItemIndex];
    text = (text || $('svReplyInput').value || '').trim();
    if (!item || !text || item.ownerId === ME.uid) return;
    try {
        const chatId = await getOrCreatePrivateChat(item.ownerId);
        await sendDirectMessage(chatId, {
            text,
            storyReplyTo: {
                storyId: item.id,
                mediaUrl: item.mediaUrl,
                ownerName: item.ownerName || ''
            }
        });
        $('svReplyInput').value = '';
        showMockModal('جواب فرستاده شد');
    } catch (e) {
        showMockModal('خطا در ارسال جواب');
    }
}

async function reactToStory(emoji) {

            storyReplyTo: {
                storyId: item.id,
                mediaUrl: item.mediaUrl,
                ownerName: item.ownerName || ''
            }
        });
    } catch (e) {}
}
```

این کل بخش استوری رو می‌بنده. جمع‌بندیِ وابستگی‌ها که همه چک شدن و درستن:

- `pushMessage` دست‌نخورده موند؛ جواب استوری از `sendDirectMessage` جدا استفاده می‌کنه تا چتِ بازِ فعلی خراب نشه.
- `getOrCreatePrivateChat` با `query` + `getDocs` روی `members array-contains` پیاده شد (همه‌ی importهاش از قبل توی فایل بودن).
- `escapeHtml` رو قبلاً دادم — اگه هنوز اضافه نکردی، حتماً بذارش وگرنه `openStoryViewers` ارور می‌ده.
- جای `DEFAULT_AVATAR` از `STORY_DEFAULT_AVA` استفاده شد چون توی فایل چنین ثابتی نبود.

تنها چیزی که از سمت تو می‌مونه: قرار دادن این تکه‌ها سر جاشون و افزودن بلوک داخل `showStoryItem()`. می‌خوای `showStoryItem` کامل و یک‌پارچه (با این تکه‌ها داخلش) رو هم برات بنویسم که فقط کپی کنی؟
